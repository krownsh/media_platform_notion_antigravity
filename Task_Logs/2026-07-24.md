- `/api/process` 改為 `requireApiAuth` 保護；瀏覽器以 Supabase Bearer JWT 決定 `user_id`，n8n API key 必須以 `MEDIA_API_KEY_USER_ID` 映射固定使用者，body `userId` 已移除。

## Agent-Driven 互動式資料工作流實作完成（2026-07-24 下午）

### 完成項目
1. **重新調整架構設計 (Pivoting)**：根據主人的安全考量，廢棄原先「背景自動執行 (Background Polling)」的危險構想，轉向實作 **Interactive Agent-Driven 工作流 (Human-in-the-loop)**。伺服器僅負責收集與存入佇列，所有執行與 POC 測試交由 Agent (如 Codex/Claude) 在終端機與使用者互動中完成。
2. **建立 Agent CLI 工具包 (`scripts/agent-sdk/`)**：
   - 實作 `get-inbox.js` (`npm run agent:inbox`)：連接資料庫撈取 `status = 'pending'` 的待處理收藏事件，提供概要列表。
   - 實作 `analyze-item.js` (`npm run agent:analyze <id>`)：封裝 `routeAgent` 與 `opportunityMatcher`，對指定收藏進行深度分類與 POC 提案。
   - 實作 `complete-item.js` (`npm run agent:complete <id>`)：讓 Agent 在完成實作或驗證後，能將資料庫中的狀態標記為 `processed`。
3. **修復 Supabase Client**：修改 `server/supabaseClient.js` 的 Mock Chain，確保在未載入 Service Key 或處於純前端環境時，Mock 客戶端能正確支援 `.eq()` 和 `.order()` 等串列操作而不會拋出 `TypeError` 導致腳本崩潰。
4. **整合 Package Scripts**：在 `package.json` 中加入 `agent:inbox`, `agent:analyze`, `agent:complete`，便於 Agent 快速呼叫。
